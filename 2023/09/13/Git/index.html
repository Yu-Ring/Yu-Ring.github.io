概述

免费开源的分布式版本控制系统

  • 本地版本控制：所有的版本保存在本地，个人使用。如RCS。
  • 集中式版本控制工具：所有的版本保存在服务器上，所有用户通过服务器获取版本。服务器发生故障时，所有上传者都不能做版本记录。如SVN。
  • 分布式本本控制工具：所有版本记录都同步到每个用户的本地，本地可完成离线版本修改，联网时通过push推送到服务器，再由其他用户同步。如Git。

工作机制

  • 工作区：本地磁盘存放代码的地方
  • 暂存区：临时存储，工作区通过git add添加到暂存区。
  • 本地库：本地生成的历史版本记录仓库。暂存区通过git commit上传到本地库
  • 代码托管中心（远程库）：本地库通过git push推送到代码托管中心。

Git命令

常用LInux操作命令

  • cd : 改变目录。
  • cd .. 回退到上一个目录，直接cd进入默认目录
  • pwd : 显示当前所在的目录路径。
  • ls ll: 列出当前目录中的所有文件，后者列出的内容更为详细。
  • touch : 新建一个文件 如 touch index.js 就会在当前目录下新建一个index.js文件。
  • rm: 删除一个文件, rm index.js 就会把index.js文件删除。
  • mkdir: 新建一个目录,就是新建一个文件夹。
  • rm -r : 删除一个文件夹, rm -r src 删除src目录
    ==rm -rf / 切勿在Linux中尝试！删除电脑中全部文件！==
  • mv 移动文件, mv index.html src index.html 是我们要移动的文件, src 是目标文件夹,当然, 这样写,必须保证文件和目标文件夹在同一目录下。
  • reset 重启终端。
  • clear 清屏。
  • history 查看命令历史。
  • help 帮助。
  • exit 退出。
  • #表示注释

常用Git命令

  • 项目创建和克隆
  • Git文件操作
    文件状态：
    Untracked ：未跟踪，文件还在文件夹中，尚未加入到git库，不参与版本控制。git add进入Staged
    Unmodify：文件已经入库，未修改。要么修改进入Modified，要么git rm移出版本库进入Untracked
    Modified：文件已修改。要么git add进入Staged，要么git checkout从库中取出返回Unmodify
    Staged：暂存状态，git commit把本地同步到库中。git reset HEAD filename取消暂存 文件忽略：gitignore
